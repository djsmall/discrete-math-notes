Set-Builder Notation
...

"The set with all the elements (x) are an element in the set (A), such that the elements are greater than 0." can be expressed as:
{ x ∈ A | x > 0 }
This is what each notation means:

  • { } - Elements within the brackets are part of a set.
  • ∈ - "Is an element of" refers to elements from another set that will be copied for this set.
  • A - the set A
  • | - "Such that", notation to the right of this are the conditions

Set Operations
...

A Set Operation are math operations done on sets in order to create new sets.

  • ⊆ - Subset, is every element in set A is also an element of set B, then A is a subset of B. Every set is a subset of itself.
  • ⊂ - Proper Subset every element in set A is found in set B, but there are additional elements in set B that are not found in set A.
    Union

    ∪ - Union of two sets, such as A ∪ B, read as "A union B". If an element is in set A or B (inclusive or), it will be in the new set.
    Formal definition: A ∪ B = { x | x ∈ A ∨ x ∈ B }

Intersection

∩ - Intersection of two sets, such as A ∩ B, read as "A intersect B". If there are elements that are in both A and B (and only if), it will be in the new set.
Formal definition: A ∩ B = { x | x ∈ A ∧ x ∈ B }

Difference

"-" - Difference of two sets, such as A - B, is the set of elements that are in A but not B.
Formal definition: A - B { x : x ∈ A and x ∉ B }

Symmetric difference

⊕ - Symmetric Difference of two sets, such as A ⊕ B, is the set of elements that are in exactly one of A and B, but not both.
Formal definition: A - B { x : x ∈ A and x ∉ B }

Functions
...

f(x) = y
A function is a relation that uniquely associates each member of one set with exactly one member of another set.
Domain - The input set of the function (without breaking it)
Co-Domain (aka Target or Range) - The output of the function
Every domain element must be connected to exactly one co-domain element. It is no longer a function if a domain element connects to multiple co-domains elements, or if a domain element does not have any connections. However, It is okay to have a co-domain element with multiple domain elements connecting to it, as well as have have "extra" elements in the co-domains that are just hanging out, unrelated to any domain elements.
